# Lecture Review Project

Frontend : Vue.js 3
역할 : 사용자 UI 구현, 강의 목록 후기 작성 등 인터페이스 담당
선정이유 : Vue.js는 컴포넌트 기반 구조로 UI를 효율적으로 개발할 수 있으며 ,학습 난이도가 비교적 낮아 팀원들이 빠르게 적응할 수 있다 또한 API 연동이 용이하여 백엔드와의 통신에 적합하다.

Backend
FastAPI: 비동기 처리 기반의 높은 성능을 제공하는 Python 웹 프레임워크로, API 문서 자동 생성 기능을 지원하여 프론트엔드와의 협업 효율성을 높이기 위해 선정하였다.
SQLModel: Pydantic과 SQLAlchemy의 장점을 결합한 ORM으로, 하나의 클래스로 데이터 검증과 데이터베이스 모델 정의를 동시에 수행할 수 있어 코드 일관성과 유지보수성을 향상시키기 위해 사용하였다.
JWT (JSON Web Token): 서버가 사용자 상태를 저장하지 않는 Stateless 인증 방식을 통해 확장성을 고려한 인증 구조를 구현하기 위해 도입하였다.
bcrypt: 사용자 비밀번호를 단방향 해싱 방식으로 저장하여, 데이터베이스 유출 시에도 원문 비밀번호를 복구할 수 없도록 하기 위해 사용하였다.

Database
Neon (PostgreSQL): 서버리스 기반의 클라우드 데이터베이스로, 별도의 인프라 구축 없이 팀원들이 동일한 데이터베이스에 접근하여 협업할 수 있는 환경을 구성하기 위해 선택하였다.

DevOps / Tools
uv: Rust 기반의 빠른 패키지 매니저로, 의존성 설치 및 가상 환경 구성을 효율화하여 개발 생산성을 향상시키기 위해 사용하였다.
Swagger UI: 자동 생성된 API 문서를 기반으로 엔드포인트를 즉시 테스트할 수 있어, 프론트엔드와의 협업 과정에서 발생할 수 있는 소통 오류를 줄이기 위해 활용하였다.
시스템 구조 : Frontend (Vue.js) → Backend (FastAPI) → Database (Neon PostgreSQL)
